# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** open a public GitHub issue
2. Email security reports to: **tiletopia@proton.me**
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Assessment**: Within 7 days
- **Fix**: Within 30 days for critical issues

## Scope

Security issues in the following areas are in scope:

- REST API authentication/authorization bypass
- Input validation (address parsing, data ingestion)
- Denial of service via crafted input
- Information disclosure
- Dependency vulnerabilities

## Recognition

We credit security researchers in our CHANGELOG unless anonymity is requested.
